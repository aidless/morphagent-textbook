# 实验 3：工具描述对调用准确率的影响

> 状态：可运行
> 目标：量化"工具描述质量对 LLM 调用准确率的影响"
> 运行时长：< 2 分钟

## 实验设计

### 5 个描述级别

| 级别 | 描述内容 | 期望准确率 |
|---|---|---|
| L0 极简 | 仅工具名 + 一句话 | ~35% |
| L1 有参数 | 工具名 + 需要的参数 | ~60% |
| L2 有参数+类型 | + 参数类型说明 | ~78% |
| L3 有示例 | + 调用示例 | ~88% |
| L4 完整 | + 返回值、错误、场景 | ~95% |

### 5 个测试工具

- get_weather（查天气）
- search（搜索）
- add（加法）
- read_file（读文件）
- send_email（发邮件）

### 10 个测试任务

覆盖每个工具的 2-3 个用例。

## 假设

- **H1**：描述完整度（L0→L4）单调提升 LLM 调用准确率。
- **H2**：从 L0 到 L4，准确率提升至少 40 个百分点。

## 指标

- **任务成功率**：完成的任务比例
- **工具+参数匹配率**：LLM 选择了正确工具且参数正确的比例
- **每个 LLM 的差异**：GPT-4 vs Claude 3.5 vs Llama 3.1

## 运行方式

```bash
cd experiments/exp-03-tool-description
python run.py
```

## 预期结果

```
| 描述级别 | 任务成功率 | 工具+参数匹配率 |
|---|---|---|
| L0 极简 | 35% | 30% |
| L1 有参数 | 60% | 55% |
| L2 有参数+类型 | 78% | 75% |
| L3 有示例 | 88% | 85% |
| L4 完整 | 95% | 92% |
```

## 已知限制

- Mock LLM 是确定性的，无法模拟真实 LLM 的不确定性
- 真实 LLM 实验需要约 $20 预算
- 没有统计显著性检验（样本量小）
- 只测试了 5 个工具，泛化性需更多基准
