# 实验 22：金丝雀部署模拟

> 状态：可运行（Mock 模式）
> 目标：模拟金丝雀发布策略，验证渐进式部署的安全性
> 难度：★★★★☆
> 预计时间：6 小时

## 实验目的

模拟金丝雀部署过程：新版本从 5% 流量开始，逐步扩大到 100%。
验证能否在流量扩大过程中检测到版本退化。

## 实验步骤

1. 定义稳定版本 v1.0 和候选版本 v2.0
2. 模拟 10 个流量扩展阶段（5% -> 10% -> ... -> 100%）
3. 在不同阶段注入故障（错误率、延迟、资源消耗）
4. 实现自动检测逻辑（错误率 > 阈值则停止扩展）
5. 对比金丝雀部署 vs 全量部署的风险

## 评测指标

- 检测到故障的时间（阶段数）
- 受影响用户比例
- 回滚成本

## 运行方式

```bash
cd experiments/exp-22-canary-deploy
python main.py
```
