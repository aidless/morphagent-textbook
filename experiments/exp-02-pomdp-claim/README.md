# 实验 2：POMDP 信念状态验证

> 状态：可运行
> 目标：验证第 2 章核心命题——LLM Agent 的"信念状态" ≈ "短期记忆"
> 运行时长：< 30 秒

## 实验设计

构造 5 个不同状态的 POMDP，每个 POMDP 包含：
- **真实世界状态**（Agent 不可直接观察）
- **Agent 观察到的信息**（通过感知通道）
- **最优行动**（基于"信念状态"应该采取的行动）

| POMDP | 真实状态 | 观察 | 最优行动 |
|---|---|---|---|
| p-1 | 门 A 锁着 | 门 A 锁 + 钥匙在桌上 | pick_up_key |
| p-2 | 门 A 开着 | 门 A 开 | go_through_door |
| p-3 | 门 A 锁 + 钥匙在口袋 | 摸到口袋有钥匙 | unlock_door |
| p-4 | 门外有狗 | 听到狗叫 | check_door_safely |
| p-5 | 朋友在客厅 | 听到客厅有声音 | greet_friend |

## 假设

- **H1**：LLM Agent 的最优行动选择与"信念状态"（即短期记忆中的内容）一致。
- **H2**：LLM 在记忆完整时行动合理性 > 90%。

## 指标

- **行动一致性**：LLM 选择的行动是否与"基于信念的最优行动"一致。
- **信念一致率**：5 个 POMDP 中，多少比例的 LLM 行动与信念一致。

## 运行方式

```bash
cd experiments/exp-02-pomdp-claim
python run.py
```

## 预期结果

信念一致率应达到 **100%**（mock LLM 是确定性的）。真实 LLM 下可能略低（80-90%）。

## 验证逻辑

1. **信念状态 = 短期记忆**：把"短期记忆"中包含的信息当作"信念状态"。
2. **最优行动 = 信念状态的最优响应**：基于信念状态推导出最优行动。
3. **LLM 行动 vs 最优行动**：如果两者一致，说明"LLM 的决策 = 信念状态的函数"。

## 已知限制

- Mock LLM 只能在关键词匹配层面验证信念一致；真实 LLM 下应进一步验证"信念状态部分缺失"的情况。
- 5 个 POMDP 样本量较小，缺乏统计显著性。
- 没有区分"信念正确"和"信念错误但行动正确"——可能存在"lucky wrong"情况。
