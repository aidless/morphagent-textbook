# 实验 5：上下文窗口利用效率

> 状态：可运行（Mock 模式）
> 目标：实现 LongLLMLingua 的简化版，验证上下文压缩对 Agent 性能的影响
> 难度：★★★☆☆
> 预计时间：4 小时

## 实验目的

实现句子重要性评分 + Top-K 选择，在长对话上验证压缩率与任务成功率的 tradeoff。

## 实验步骤

1. 实现 `sentence_importance()` 函数：基于关键词匹配和位置加权评分
2. 实现 `compress_context()` 函数：按 Top-K 选择高重要性句子
3. 构造 5 个长上下文对话（500-1000 token）
4. 在不同压缩率（50%/70%/90%）下测试任务成功率
5. 对比压缩前后的性能

## 评测指标

- **压缩率**：压缩后 token / 原始 token
- **任务成功率**：压缩后能否回答问题
- **关键信息保留率**：关键句子是否被保留

## 预期结果

| 压缩率 | 预期成功率 | 预期关键信息保留 |
|---|---|---|
| 50% | ~85% | ~90% |
| 70% | ~75% | ~70% |
| 90% | ~55% | ~50% |
