2023 年 AutoGPT 刷屏的时候，社区有个笑话："给 LLM 一根足够长的 prompt 链条，它就能管理公司。"三年后回头看，这句话一半荒谬，一半预言——荒谬在于 prompt 链条不够用，预言在于 Agent 确实在做一件以前只有生物才能做的事：**修改自己**。

OPRO 让 LLM 自己搜索最优 prompt，Voyager 让 Minecraft Agent 自动写技能库，SICA 让 Agent 自我修改代码提升生成质量，MemGPT 让 Agent 管理自己的记忆分页。这些工作散落在不同团队和场景中，指向同一个现象：**LLM Agent 正在获得"身体可塑性"**。

但研究界缺少一个统一框架来回答：当 Agent 能修改自己的 prompt、工具、记忆和代码，它本质上是什么系统？

《操作形态学：自修改 LLM 智能体的具身认知》这本书给出了一个答案。

## 核心框架

书里提出**操作形态（Operational Morphology）**概念，把 LLM Agent 的"软件身体"定义为四个可写组件：

- **P（Prompt）**：系统提示词、任务指令
- **T（Tool）**：可调用的外部函数和 API
- **M（Memory）**：长期记忆、经验库、检索索引
- **C（Code）**：Agent 自身执行的代码

这四个组件构成一个四元组 **B = {P, T, M, C}**。谁来修改它？一个**元控制器 U**，在每一轮交互后基于环境反馈和约束集决定下一轮的形态。

关键洞察：这四个组件不是独立的可优化变量，而是**耦合的形态系统**。修改 prompt 可能需要新工具支持，增加记忆可能需要重构代码。这种跨组件的协同进化才是自修改 Agent 区别于简单自动调参的核心。
